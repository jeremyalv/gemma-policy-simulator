# Release Procedure

## Goal
Ship policy changes safely with clear operational context.

## Workflow
1. Summarize behavior changes and impacted modules.
2. Validate check/test status and open risks.
3. Provide rollout plan and rollback trigger conditions.
4. List required monitoring signals post-release.

## Release Notes Template
- Changed behavior
- Affected inputs/policies
- Risk and mitigation
- Rollback instructions
